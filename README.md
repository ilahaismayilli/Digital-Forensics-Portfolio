🔍 Project Overview
This project involved performing a forensic investigation of a directory traversal attack on an FTP server. Using captured network traffic, system artifacts, and Linux command‑line tools, I analyzed attacker behavior, identified compromised files, and documented evidence following professional incident‑response methodology.

🧰 Tools & Techniques
Wireshark — PCAP analysis, packet filtering, protocol inspection

Linux CLI & Bash — log review, file integrity checks, attack script execution

Digital Forensics Methods — evidence documentation, chain of custody, timeline reconstruction

Incident Response Analysis — mapping file changes to attack vectors

🧩 Investigation Process
Analyzed PCAP files to isolate suspicious FTP sessions.

Identified directory traversal patterns (../) indicating unauthorized access.

Correlated timestamps between network traffic and file modifications.

Mapped compromised files to specific attacker actions.

Documented findings with screenshots, logs, and structured reporting.

📈 Key Findings
The attacker exploited weak FTP permissions to access restricted directories.

Multiple files were accessed or modified outside the intended scope.

Network forensics revealed attacker IP, session behavior, and traversal attempts.

Evidence confirmed a clear pattern of directory traversal exploitation.

This project strengthened my ability to:

Perform network and system forensics in a controlled lab environment.

Produce clear, professional incident reports suitable for SOC or DFIR roles.

Understand how directory traversal vulnerabilities expose sensitive data.

It directly supports my Digital Forensics degree by demonstrating practical skills in evidence analysis, investigative methodology, and technical documentation.
